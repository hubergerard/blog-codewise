Pourquoi utiliser le pipe async ?

Lorsque vous utilisez les promesses et les observables dans vos composants angular vous êtes soumis à vous servir de then et de subscribe pour les exécuter. Mais grâce au pipe async, vous pouvez afficher le résultat sur le template et plus besoin de then et de subscribe !

Exemple observable sans pipe async :

{% highlight ts linenos %} export class ObservableComponent implements OnInit { result; constructor() { const monObservable = of('Hello World'); monObservable.subscribe(data => this.result = data); } ngOnInit(): void {} } {% endhighlight %}
observable.component.ts

Pour l'exemple on a utilisé la fonction of de rxjs qui permet de créer un observable manuellement, la plupart du temps vous recupérerez un observable.

{% highlight ts linenos %} {% raw %}{{ result }}{% endraw %} {% endhighlight %}
observable.component.html

Testez ce code , et voyez que Hello World s'affiche bien côté html.

Exemple promise sans pipe async :

{% highlight ts linenos %} export class PromiseComponent implements OnInit { data; constructor() { const maPromesse = Promise.resolve('Hello World'); maPromesse.then(res => this.data = res); } ngOnInit(): void { } {% endhighlight %}
promise.component.ts
{% highlight ts linenos %} {% raw %}{{ data }}{% endraw %} {% endhighlight %}
promise.component.html

Maintenant il y a une manière encore plus simple d'afficher un observable et une promesse c'est ce que vous allez voir ci-dessous.

Lier un observable à la vue

Précedemment vous avez vu comment afficher un observable, mais maintenant vous allez voir qu'angular fournit une manière plus propre d'afficher un observable sans subscribe grâce au pipe async !

Démonstration :

{% highlight ts linenos %} export class ObservableComponent implements OnInit { monObservable; constructor() { this.monObservable = of('Hello World'); } ngOnInit(): void {} } {% endhighlight %}
observable.component.ts
{% highlight ts linenos %} {% raw %}{{ monObservable | async }}{% endraw %} {% endhighlight %}
observable.component.html

Tout simplement ! Cela a l'avantage d'être moins verbeux et d'éviter des erreurs.

Lier une promesse à la vue

Même principe pour les promesses, plus besoin d'utiliser then, le pipe async le fait à notre place.

{% highlight ts linenos %} export class PromiseComponent implements OnInit { maPromesse; constructor() { this.maPromesse = Promise.resolve('Hello World'); } ngOnInit(): void {} } {% endhighlight %}
promise.component.ts
{% highlight ts linenos %} {% raw %}{{ maPromesse | async }}{% endraw %} {% endhighlight %}
promise.component.html

Conclusion

Voilà c'est tout pour le pipe async, bien entendu comme expliqué plus haut, la déclaration d'observable et de promesse dans les exemples sont fait de manière manuelle, alors que la plupart du temps vous les récupérerez directement.
