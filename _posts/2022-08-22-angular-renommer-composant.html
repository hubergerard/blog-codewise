Introduction

Comme vous avez pu le constatez il n'y a pas de commande pour pouvoir renommer un composant alors vous avez deux options :

  • Renommer manuellement le composant dont tous ses fichiers.
  • Ou l'utilisation d'un plugin (sous VS CODE)

Renommer un composant manuellement

La façon manuelle est assez rébarbative car il faut renommer tous les fichiers et les noms des classes qui les composent.

Prenez cet exemple :

Dossier du composant map-filter

Vous devez renommer le composant map-filter en map. Pour cela renommer le dossier et les fichiers css, html, ts et spec.ts.

Ce qui devrait vous donner ça :

Dossier du composant renommé

Comme vous pouvez le voir l'IDE nous prévient de certaines erreur en rouge dû au renommage de nos fichiers.

Erreur des chemins fichier ts

Il va falloir donc mettre à jour les chemins qui correspondent aux fichier css et html.

Bien sûr vous pouvez modifier le nom des classes pour qu'elles correspondent aux noms des fichiers ainsi que le sélecteur.

Renommer les chemins et classes

Ensuite il vous faudra donc mettre à jour l'import du fichier ts dans le fichier spec.ts en renseignant le nouveau nom donné à la classe.

Idem pour le fichier app-routing.module.ts si votre composant est reliée a une route et surtout n'oubliez pas le app.module.ts qui lui se charge d'injecter votre composant !

Renommer un composant avec un plugin

Récemment j'ai découvert un petite extension VS Code qui permet de faire tout ce travail rapidement !

Extension VS Code pour renommer un composant Angular

Cherchez la avec la barre de recherche dans l'onglet extension de VS Code et installez la.

Sur la page de l'extension il y a une petite vidéo qui explique comment s'en servir, c'est très simple, cliquez droit sur l'un des fichiers concernant le composant à renommer alors une fenêtre s'ouvrira pour écrire le nouveau nom et appuyez sur entrer.

Il modifiera automatiquement tous les fichiers et même les noms des classes, toutefois attention vérifiez vos fichier app-routing et app.module !

Conclusion
