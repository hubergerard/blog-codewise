Impossible de connaître tous les frameworks, et difficile de savoir rapidement lequel est approprié à son projet. C’est pourquoi nous vous avons rédiger cet article afin de vous orienter autour de trois des frameworks front les plus populaires, pour passer en revue leurs avantages, inconvénients et leurs caractéristiques propre.

Tableau des critères

Description des critères

  • Maintenabilité : Facilité à évoluer dans le temps
  • Scalabilité : Facilité à s’adapter à de grands projets
  • Apprentissage sur long termes : Il sera plus intéressant de choisir un framework où l’on aura moins besoin de se remettre à jours au niveau de ses connaissances.
  • Performances : temps de réponses du framework (mesures + articles)
  • Communauté : Nombres repository github et threads Stackoverflow

Vue.js

Présentation

Vue.js est le framework javascript créer en 2013 par Evan YOU, le plus simple à appréhender de la liste. Aussi appelé « Vue », l’un des gros avantages de ce framework est sa simplicité car il offre un choix réduit. Selon votre niveau d’expérience il s’adapte à tout type de niveau que sa sois pour les débutants ou les développeurs avancés. Le templating et sa stylisation sont en HTML, CSS ce qui vous fera gagner pas mal de temps d’adaptation. Vue permet d’étendre le HTML avec des attributs HTML appelés : « directives ». Les composants Vue encapsule le HTML pour rendre le code réutilisable. Un point important est que Vue a pour notion le DOM virtuel comme dans d’autres frameworks javascript. Cette notion est importante car l’intérêt est d’éviter de mettre le DOM d’origine à jour à chaque modification.

Inconvénients

  • - Le plus récent des framworks et donc le moins optimisé, quelques problèmes d’intégration parfois.
  • - Une communauté beaucoup plus faible que les autres frameworks.
  • - N’intègre pas tous les outils nécessaires à la création d’une application front.

Avantages

  • - S’Adapte en fonction de vos compétences
  • - Relativement Simple
  • - HTML, CSS

React.js

Présentation

ReactJs, qu’on appelle aussi React, est un framework JavaScript, créée par Facebook en 2013. Il permet aussi de concevoir des interfaces utilisateurs web via la création de composants dépendants. React intègre aussi la notion de DOM virtuel. Très flexible et plus performant que d’autres framework il reste un outil apprécié par les développeurs, avec une communauté et une équipe solide la développant. Le templating est recommandé avec JSX. Les caractéristiques de React sont les composants et le flux de données unidirectionnel.

Inconvénients

  • - Templating en JSX et non en HTML
  • - Manque de documentations
  • - Offre beaucoup de choix aux développeurs : pas toujours confortable.
  • - Demande des bibliothèques externes pour la réalisation d’une application complète.

Avantages

  • - Composants et modèle de données
  • - Open source il est mis à jour régulièrement et s’adapte aux contraintes actuelles.
  • - Utilisation simple des composants

Angular

Présentation

Un cadre des framework javascript les plus populaires et le plus utilisés pour créer des applications côté utilisateur. Angular n’est pas nouveau, il existe depuis un certain temps AngularJs était la première version et était très différente de l’actuelle ! En effet Angular est dotée de beaucoup plus d’outils et de fonctionnalité que la précédente version. Par exemple, l’injection de dépendance, liaisons de données, tests unitaires etc… Ce qui rend Angular a tout type de tâche de développement en amont. Très flexible en termes de développement, il répond aux besoins de : composants (d’encapsulation de fonctionnalités), le templating, le routing, gestions d’erreurs, de communiquer avec l’API.

Inconvénients

  • - Apprentissage long et ardu ce qui peut être décourageant dans un premier temps.
  • - Tellement complet qu’il peut paraître assez complexe.

Avantages

  • - Grosse communauté
  • - Bonne documentation
  • - HTML, CSS pris en charge
  • - Robuste et performant

Conclusion

Après avoir vu en globalité les 3 frameworks javascript les plus populaire du moment, on se demanderai presque toujours lequel choisir ! la réponse est que chaque framework est utile pour un type précis de projet. Mais en analysant les informations on peut remarquer qu’Angular dispose de tous les outils nécessaires et fonctionnalités qu’une interface d’application pourrait avoir besoin. Bien sûr l’inconvénient est qu’il donc plus difficile à appréhender. React et Vue sont plus simple mais vous allez aussi vous rendre compte que : - il vous faut du routing, - que votre application grossit, - que vous n’avez pas d’architecture, - que vous n’avez pas de gestion d’erreur, - etc… Vue est avant tout un bon choix pour transformer un site web existant en application d’une manière progressive. Finalement vous devrez ajouter tout cela avec des heures de configurations à la clé. Le temps gagné dans l’apprentissage est peut-être perdu dans le manque d’outils ?
