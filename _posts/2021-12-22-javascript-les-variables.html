Qu’est-ce qu’une variable ? Pour les débutants en programmation nous allons découvrir ce sujet avec le langage javascript !

Historique

Tout d’abord un peu d’histoire, le javascript a été créé en 1995 en seulement 10 jours par Brendan Eich. A la base son nom d’origine est Mocha qui a évolué en Livescript puis en Javascript comme nous le connaissons. À partir de 1996 Javascript est supporté par tous les navigateurs. En 1997 le langage est standardisé sous le nom ECMAScript. Javascript est une implémentation d’ECMAScript (ES) (la version actuelle est la 8ème : ES8)

Définition

Et donc qu’est-ce qu’une variable ? En informatique une variable est un élément de code qui fait référence à une position de mémoire qui contient de la donnée et qui peut successivement prendre différentes valeurs au cours de l’exécution d’un programme. Nous verrons des exemples un peu plus tard dans cet article.

Convention de nommage

En tant que développeur ou futur développeur, nous sommes soumis à des conventions pour nommer nos variables, pourquoi ? Pour une question de lisibilité de code et de maintenabilité.

Un nom de variable doit obligatoirement commencer par une lettre (minuscule ou majuscule), un ‘$’ ou un ‘_’.

Un nom de variable peut aussi contenir des chiffres, et les caractères spéciaux suivants : ‘$’ et ‘_ ‘.

On rappellera que les espaces sont interdits dans le nom d’une variable (cela produira forcément une erreur).

Ensuite on choisira un nom en anglais qui donnera du sens à la variable, par exemple pour garder mon prénom dans le programme j’écrirais : firstName.

Vous remarquerez qu’il n’y a pas d’espace et qu’à partir du deuxième mot on utilise une majuscule. Cette convention de nommage s’appelle lower camelCase, le tout premier mot commence toujours par une minuscule et les suivants par une majuscule.

Déclaration

Une variable doit obligatoirement être déclarée et également il est conseillé de l’initialiser avec une valeur par défaut.

< clé > < nom de la variable > = [valeur par défaut] ;

let firstName = "Pierre" ;

Voici les différents mots-clés pour déclarer une variable :

  • Var (ES5, déprécié)
  • let
  • const

En fonction du mot-clé, la portée et l’utilisation de la variable peuvent changer.

L’assignation

Pour assigner une valeur à une variable, on utilise l’opérateur d’affectation : =

On peut le faire lors de la déclaration de la variable comme dans l’exemple dans le chapître précédent, on appelle cela l’initialisation. On peut par la suite assigner une nouvelle valeur à une variable qui écrasera la valeur précédente. Et on peut aussi utiliser la valeur courante d’une variable pour calculer une nouvelle valeur à lui assigner.

Exemple d’assignation :

{% highlight js linenos %} let age = 25; // age value is 25 age = 26; // age value is 26 age = age + 1 // age value is 27 {% endhighlight %}

Type d’une variable

En javascript comme dans différents langages, nous manipulons plusieurs types de données :

  • Des nombres (Numbers), entier ou à virgule.
  • Des chaînes de caractères (String).
  • Des tableaux (Array).
  • Des objets (Object).
  • Des booléens (Boolean), représentant des variables permettant de vérifier des conditions (true ou false).

On peut connaître le type d’une variable avec typeOf, exemple :

{% highlight js linenos %} const firstName = "John"; typeof firstname; // => "string" const age = 25; typeof age; // => "number" const user = { firstName : "John", age : 25 } typeof user; // => "object" {% endhighlight %}

Portée d’une variable

Une variable est obligatoirement déclarée à l’intérieur d’un bloc d’instruction. Elle est connue jusqu’à la fin du bloc d’instruction.

Élément syntaxique d’une instruction :

Règle d’indentation : les instructions mises entre accolades sont indentées.

Cas exceptionnel : une variable déclarée avec le mot-clé var, sa portée est limitée à la fonction dans laquelle elle est déclarée.

Exemple avec let :

{% highlight js linenos %} // the reference is not defined before the declaration, it throws an error console.log(name); // => ReferenceError : name is not defined let name = "John"; // it defines a new scope { . . . } if (name === "John") { let age = 25; console.log(age); // => 25 } console.log(name); // => "John" // the reference is not defined in this scope, it throws an error console.log(age); // => // => ReferenceError : name is not defined {% endhighlight %}

Exemple avec var :

{% highlight js linenos %} // the reference is defined but the value is not defined (hoisting) console.log(name); // => undefined var name = "John"; // it defines a new scope { . . . } if (name === "John") { var age = 25; console.log(age); // => 25 } console.log(name); // => "John" // age was defined in another scope but still exists in this scope console.log(age); // => 25 {% endhighlight %}

Constante

Comme son nom l’indique elle n’est pas variable mais constante ! Une constante est déclarée avec le mot-clé const, elle ne peut être initialisée qu’une fois et ne peut pas être réassignée d’une autre valeur. Toutes les variables dont les valeurs ne sont pas réassignées devraient être déclarées avec const pour des raisons de performance et simplicité de lecture du code

{% highlight js linenos %} const favoriteSchool = "CodeWise"; favoriteSchool = "Another School"; // => TypeError: Assignement to constant variable. {% endhighlight %}
