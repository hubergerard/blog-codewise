Introduction

Les paramètres de routes sont utilisés dans Angular pour transmettre des données entre les différentes routes de votre application.

Cela vous permet de créer des URL dynamiques et de passer des valeurs spécifiques à une route donnée.

Les paramètres de routes sont souvent utilisés pour afficher des détails spécifiques d'une ressource ou pour filtrer les données affichées.

Définition des routes

Tout d'abord, vous devez définir vos routes dans le fichier de configuration des routes de votre application, généralement appelé app-routing.module.ts. Vous pouvez utiliser le RouterModule d'Angular pour définir vos routes.

Par exemple :

{% highlight ts linenos %} import { NgModule } from '@angular/core'; import { RouterModule, Routes } from '@angular/router'; import { HomeComponent } from './home.component'; import { DetailsComponent } from './details.component'; const routes: Routes = [ { path: '', component: HomeComponent }, { path: 'details/:id', component: DetailsComponent }, ]; @NgModule({ imports: [RouterModule.forRoot(routes)], exports: [RouterModule] }) export class AppRoutingModule { } {% endhighlight %}
App-routing.module.ts

Dans cet exemple, nous avons défini deux routes : une route racine (path: '') qui correspond au composant HomeComponent, et une route details/:id qui correspond au composant DetailsComponent.

La partie :id dans la deuxième route indique qu'il s'agit d'un paramètre dynamique qui peut prendre différentes valeurs.

Récupération des paramètres de route

Une fois que vous avez défini vos routes, vous pouvez accéder aux paramètres de route dans votre composant en utilisant le service ActivatedRoute fourni par Angular. Vous devez injecter ce service dans votre composant.

Voici un exemple :

{% highlight ts linenos %} import { Component, OnInit } from '@angular/core'; import { ActivatedRoute } from '@angular/router'; @Component({ template: `

Details Component

ID: {{ id }}

` }) export class DetailsComponent implements OnInit { id: string; constructor(private route: ActivatedRoute) { } ngOnInit() { this.route.params.subscribe(params => { this.id = params['id']; }); } } {% endhighlight %}
Details.component.ts

Dans cet exemple, nous injectons le service ActivatedRoute dans le constructeur du composant et utilisons la méthode params.subscribe() pour écouter les changements des paramètres de route.

Lorsque les paramètres de route changent, la fonction de rappel est appelée, et nous pouvons récupérer la valeur du paramètre id à l'intérieur.

Navigation avec des paramètres de route

Pour naviguer vers une route avec des paramètres, vous pouvez utiliser le service Router d'Angular.

Par exemple :

{% highlight ts linenos %} import { Component } from '@angular/core'; import { Router } from '@angular/router'; @Component({ template: ` ` }) export class HomeComponent { constructor(private router: Router) { } goToDetails(id: number) { this.router.navigate(['/details', id]); } } {% endhighlight %}
Home.component.ts

Dans cet exemple, nous avons un bouton dans le composant HomeComponent qui déclenche la fonction goToDetails() lorsqu'il est cliqué.

Cette fonction utilise le service Router pour naviguer vers la route details/:id en passant l'ID en tant que paramètre.

Conclusion

C'est ainsi que vous pouvez utiliser les paramètres de routes dans Angular pour transmettre des données entre les différentes routes de votre application. Les paramètres de routes sont un outil puissant pour créer des applications dynamiques et interactives.
