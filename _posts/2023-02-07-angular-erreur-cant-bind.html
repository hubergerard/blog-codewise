Introduction à l'erreur : "Can't bind to"

L'erreur Can't bind to 'x' since it isn't a known property of 'y' peut survenir de plusieurs causes, elle est assez récurente dans Angular et peut se produire lorsqu'une directive, un composant ou un élément HTML n'a pas accès à une propriété déclarée dans le modèle ou déclarée de manière incorrect.

Mais vous allez voir cela en détails dans la suite de cet article.

Comment cette erreur est provoquée ?

Syntaxe incorrect

Une mauvaise syntaxe peut provoquer cette erreur, par exemple si le nom de la directive, du composant, du pipe ou du module dans le fichier typescript ou html est incorrect.

Pour cela, assurez vous que la syntaxe de votre élément est correct dans le composant où il est appelé exemple :

Problème d'import

Une deuxième cause serait l'oubli d'importation du module dans votre fichier app.module.ts dont vous essayer d'utiliser la directive ou autre élément dans votre html exemple :

{% highlight html linenos %}
{% endhighlight %}
your.component.html
{% highlight ts linenos %} import { Component } from '@angular/core'; import { FormGroup, FormControl, Validators, FormBuilder } from '@angular/forms'; import {FormsModule,ReactiveFormsModule} from '@angular/forms'; import { Task } from './task'; @Component({ selector: 'app-your', templateUrl: 'app/task-add.component.html' }) export class yourComponent { newTaskForm: FormGroup; constructor(fb: FormBuilder) { this.newTaskForm = fb.group({ name: ["", Validators.required] }); } createNewTask() { console.log(this.newTaskForm.value) } } {% endhighlight %}
your.component.ts

Vous ne remarquez rien d'anormal jusqu'ici mais l'erreur est présente et pourtant les modules sont bien importé dans le composant.

En réalité, l'erreur provient du fait que les modules n'ont pas été importé dans le fichier app.module.ts :

{% highlight ts linenos %} import { NgModule } from '@angular/core'; import { BrowserModule } from '@angular/platform-browser'; import { FormsModule, ReactiveFormsModule } from '@angular/forms'; import { AppComponent } from './app.component'; @NgModule({ imports: [ BrowserModule, FormsModule, ReactiveFormsModule ], declarations: [ AppComponent, yourComponent ], bootstrap: [AppComponent] }) export class AppModule { } {% endhighlight %}
your.component.html

Bonnes pratiques

  • Vérifier l'import du module dans le fichier app.module.ts pour chaque directive importé dans votre composant
  • Utiliser un IDE qui permet de mettre en surbrillance les référence non importé et les erreurs syntaxique

Conclusion

En fin de compte pour résoudre cette erreur, vous devez vous assurer que la référence que vous souhaitez utiliser est importée et déclarée correctement dans le composant et dans le fichier app.module.ts, et que le nom de la référence est correctement appelé syntaxiquement dans le html.

Si la propriété ou référence est déclarée dans un autre module, assurez-vous qu'elle est correctement exportée et importée dans le composant.
