Définition

Un objet c'est un ensemble de propriétées (ou d'attributs), et de méthodes cohérentes entre elles.

Regrouper des données

Nous avons découvert précédemment les tableaux, qui permettent de regrouper des données dans une même structure.

Dans un tableau, les données peuvent être de types différents (nombres, chaînes, booléens, etc.).

{% highlight js linenos %} const mixedTypes = [1,2, true, "string"]; {% endhighlight %}

Maintenant, admettons qu’on veuille regrouper des données sur une personne. On pourrait être tenté de le faire dans un tableau, e n indiquant, dans l’ordre : le prénom, le nom, l’année de naissance, la Seulement, voilà : chaque donnée est accessible par un index numérique… pour obtenir la nationalité, il faut utiliser personData[3]. Pas très pratique, car il faut mémoriser, par coeur, à quelle donnée correspond chaque indice !

{% highlight js linenos %} const person = ["Simon", "Dupont", 1987, "French"]; console.log(person[3]); // Affiche la nationnalité {% endhighlight %}

Les objets littéraux

Les objets offrent une autre façon de regrouper des données, au sein d’une même structure. Si les tableaux permettent d’accéder aux données via une “clé” numérique (l’index, un nombre entier), les objets permettent d’y accéder via une “clé” de type chaîne.

Voici comment on pourrait stocker les données d’une personne dans un objet, et accéder à sa nationalité :

{% highlight js linenos %} const personObject = { firstname: "Simon", lastName: "Dupont", birthYear: 1987, nationality: "French" }; console.log(personObject.nationality); // Affiche la nationnalité {% endhighlight %}

Déclaration d'objet

Il y a plusieurs manière de déclarer un objet mais nous allons nous attarder sur la création d'objet littéral :

un objet vide :

{% highlight js linenos %} const emptyObject = {}; {% endhighlight %}

un objet avec des attributs et méthodes :

{% highlight js linenos %} let student = { name : ['Simon','Dupont'], age : 22, mail : 'simon.dupont@gmail.com', bonjour: function(){ alert('Bonjour, je suis ' + this.name[0] + ", j'ai " + this.age + ' ans'); } } {% endhighlight %}

Le nommage des clés est moins contraint que celui des variables : on peut utiliser comme clé une chaîne, un nombre, un booléen, mais la clé sera de toute façon convertie en chaîne.

On parle d'objet littéral car chacune de ses propriétés et de ses méthodes a été défini lors de sa création. Par abus de langage on confondra l'objet et la variable qui le stock en citant : "l'objet".

Acceder aux propriétés d'un objet

Le caractère point : "." nous permet d'accéder aux propriétés de l'objet, on appel cela un accesseur.

{% highlight js linenos %} let student = { name : ['Simon','Dupont'], age : 22, mail : 'simon.dupont@gmail.com', bonjour: function(){ console.log('Bonjour, je suis ' + this.name[0] + ", j'ai " + this.age + ' ans'); } } console.log(student.name[0]); console.log(student.age); student.bonjour(); // expected output : // "Simon" // 22 // Bonjour, je suis Simon j'ai 22 ans {% endhighlight %}

Grâce à cela nous pouvons aussi modifier les attributs :

{% highlight js linenos %} let student = { name : ['Simon','Dupont'], age : 22, mail : 'simon.dupont@gmail.com', bonjour: function(){ console.log('Bonjour, je suis ' + this.name[0] + ", j'ai " + this.age + ' ans'); } } student.age = 23; student.displayAge = function(){ console.log(student.age); } student.displayAge(); // expected output : // 23 {% endhighlight %}

Et enfin nous pouvons aussi en définir de nouveaux !

{% highlight js linenos %} let student = { name : ['Simon','Dupont'], age : 22, mail : 'simon.dupont@gmail.com', bonjour: function(){ console.log('Bonjour, je suis ' + this.name[0] + ", j'ai " + this.age + ' ans'); } } student.notes = [12,13,9,16]; console.log(student); // expected output : // {name: ["Simon", "Dupont"], age: 22, mail: "simon.dupont@gmail.com", bonjour: function, notes: [12, 13, 9, 16]} {% endhighlight %}

On voit bien que la nouvelle propriété notes à bien été ajouté à l'objet student.

Il y a aussi une autre méthode pour accéder aux propriété d'un objet, c'est une syntaxe qui utilise les crochets et qui par contre ne permet pas les mêmes choses que la méthode précédente, car dans celle-ci nous ne pourrons pas modifier ou définir de méthode à notre objet.

Donc grâce aux crochets nous allons pouvoir utiliser notre objet comme si il était un tableau assosiatif dont les clés sont les noms des propriétés et les valeurs sont les valeurs associés.

Par exemple pour accéder a la valeur complète de la propriété name de l'objet on écrira : student['name'], pour accéder à la valeur du premier élément du tableau name : student['name'][0]

Cette syntaxe fonctionne aussi pour l'affectation et la création d'attributs au sein d'un objet.

{% highlight js linenos %} let student = { name : ['Simon','Dupont'], age : 22, mail : 'simon.dupont@gmail.com', bonjour: function(){ console.log('Bonjour, je suis ' + this.name[0] + ", j'ai " + this.age + ' ans'); } }; student['notes'] = [12,13,9,16]; student['age'] = 23; console.log(student); // expected output : // {name: ["Simon", "Dupont"], age: 23, mail: "simon.dupont@gmail.com", bonjour: function, notes: [12, 13, 9, 16]} {% endhighlight %}

Le mot-clé this

Au sein de la méthode bonjour on peut remarquer ce fameux mot-clé this, il apparaît fréquemment dans les langages orientés objets. Dans le cas présent, il fait référence à l'objet qui est manipulé donc entre autre cela veut dire : prend l'attribut name de cet objet (student) qui sera donc Simon.

Lorsqu'on écrira student.bonjour(); le mot-clé this dans la méthode sera remplacé par student au moment de l'éxécution.

Vérifier l'existence d'une propriété

Nous avons plusieurs choix qui s'offre à nous pour vérifier l'existence d'une propriété sur un objet.

Avec key in obj :

Avec obj.hasOwnProperty(key) :

{% highlight js linenos %} console.log(simon.hasOwnProperty('lastName')); // true console.log(john.hasOwnProperty('lastName')); // false {% endhighlight %}

Avec obj[key] !== undefined:

{% highlight js linenos %} console.log(simon['lastName'] !== undefined); // true console.log(john['lastName'] !== undefined); // false {% endhighlight %}

Supprimer une propriété

{% highlight js linenos %} delete obj['firstName']; delete obj.lastName; {% endhighlight %}
